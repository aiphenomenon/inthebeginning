## Turn 1: Implement RadioEngineV11

### Request
Implement the V11 radio engine based on the comprehensive code review
(19 issues identified across gain staging, consonance, beat alignment,
orchestration, and effects chain).

### Actions Taken

1. **Created 5 new classes**:
   - `_soft_limit()`: Soft-knee limiter function replacing `math.tanh`
   - `GainStage`: Per-voice RMS normalization + master bus limiting
   - `ConsonanceEngine`: Inter-voice consonance scoring and adjustment
   - `BarGrid`: Absolute time grid for bar/beat alignment
   - `OrchestratorV11`: Orchestral role assignment with voice leading
   - `RadioEngineV11(RadioEngineV10)`: Full V11 engine subclass

2. **Key fixes implemented**:
   - **Issue 7/8/9 (HIGH)**: Separated pan and gain in `_mix_mono_v11` — pan is now
     clamped to [-1, 1] and gain is a separate parameter. This eliminates
     the phase inversion caused by `pan * v_gain * 4` exceeding [-1, 1].
   - **Issue 10 (MEDIUM)**: Moved compression AFTER reverb to prevent pumping
   - **Issue 12 (MEDIUM)**: Raised minimum MIDI note from 24 to 36 (C2) to prevent
     subsonic fundamentals being stripped by the filter
   - **Issue 15 (MEDIUM)**: Increased anti-click fades from 2ms to 5ms
   - **Reverb resonance**: Reduced comb filter feedback from 0.70-0.75 to 0.55-0.65
   - **Reverb wet**: Reduced from 0.30 to 0.20
   - **Pre-reverb highpass**: Added 150 Hz highpass before reverb input
   - **Early reflections**: Reduced amplitudes from 0.08-0.35 to 0.05-0.20
   - **Crossfade duration**: Extended from 2-4s to 4-6s
   - **Bar-aligned sections**: Rondo sections now start at bar boundaries via BarGrid
   - **Inter-voice consonance**: ConsonanceEngine scores and adjusts composite harmony
   - **Orchestral roles**: Foundation/harmony/melody/color assignment with distinct
     register offsets and gain weights
   - **Voice leading**: Smooth transitions between chords (min-movement algorithm)
   - **Streaming renderer**: Uses `_soft_limit` instead of `math.tanh`

3. **Updated CLI**: Added `--version v11` option

4. **Added tests**: 25 new test methods across 7 test classes:
   - TestSoftLimit (3 tests)
   - TestGainStage (3 tests)
   - TestConsonanceEngine (5 tests)
   - TestBarGrid (5 tests)
   - TestOrchestratorV11 (3 tests)
   - TestRadioEngineV11 (5 integration tests)
   - TestV11CLIArgument (1 test)
